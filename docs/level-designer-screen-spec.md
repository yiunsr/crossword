# 레벨디자인 프로그램 화면 항목 명세서

## 문서 목적
이 문서는 한국어 십자말 풀이의 `stage.xml` 파일을 생성하기 위한 **레벨디자인 프로그램**의 화면 구성과 입력/출력 항목을 정의한다.

이 프로그램의 목적은 퍼즐 배치를 직접 설계하는 것이 아니라,

- 스테이지 조건 입력
- 필수 단어 4개 추천
- 추천 결과 검토
- `stage.xml` 생성

을 빠르고 안정적으로 수행하도록 돕는 것이다.

---

## 프로그램 역할 요약
레벨디자인 프로그램은 다음 역할을 가진다.

1. `사전.v0001.csv`를 읽는다.
2. 디자이너가 입력한 조건에 맞는 필수 단어 4개를 추천한다.
3. 추천 단어가 실제로 **2개 그룹으로 교차 가능**한지 검증한다.
4. 검토가 완료되면 `stage.xml`을 생성한다.

---

## 전체 화면 구조
화면은 3개 패널로 구성한다.

- **왼쪽 패널**: 스테이지 설정
- **가운데 패널**: 필수 단어 추천 결과
- **오른쪽 패널**: 자동 검증 및 `stage.xml` 요약

---

# 1. 왼쪽 패널: 스테이지 설정

## 1.1 스테이지 번호
- 항목명: `스테이지 번호`
- 입력 방식: 숫자 입력
- 예시값: `025`
- 역할:
  - `stage.xml`의 `level` 또는 `id` 생성에 사용
  - stage 파일명 생성 기준값으로 사용

### 제약
- 정수만 허용
- 최소값: `1`
- 데모 범위에서는 `1 ~ 100` 사용 권장

---

## 1.2 Seed
- 항목명: `Seed`
- 입력 방식: 숫자 입력
- 예시값: `27`
- 역할:
  - 랜덤 단어 선택
  - connector word 선택 순서
  - 퍼즐 생성 시도 순서
  에 영향을 주는 초기값

### 제약
- 허용 범위: `0 ~ 100`
- 데모 단계에서는 이 범위를 유지

---

## 1.3 보드 크기
- 항목명: `보드 크기`
- 입력 방식: 숫자 입력 2개
  - `행(rows)`
  - `열(columns)`
- 예시값:
  - `행 10`
  - `열 10`
- 역할:
  - 스테이지 퍼즐 생성에 사용할 보드 크기 지정

### 제약
- 각 값은 정수
- 데모 기준 권장값
  - `10 x 10`
  - `10 x 11`
  - `11 x 11`

---

## 1.4 플레이 단어 수
- 항목명: `플레이 단어 수`
- 입력 방식: 숫자 입력 1개
- 예시값: `9`
- 역할:
  - 실제 스테이지에서 사용되는 전체 단어 수의 기준값

### 자동 규칙
입력값은 목표값으로만 사용한다.

예:
- 입력값 `9`
- 내부 적용 범위 `8 ~ 10`

즉 시스템은 자동으로 다음을 계산한다.

- `min = target - 1`
- `target = 입력값`
- `max = target + 1`

### 이유
- 사용자가 범위를 직접 입력하지 않도록 단순화
- 실제 퍼즐 생성 시 유연성 확보
- 너무 빡빡한 조건으로 인한 생성 실패 감소

---

## 1.5 사용 카테고리
- 항목명: `사용 카테고리`
- 입력 방식: 다중 선택 체크박스
- 카테고리 목록:
  - `일상`
  - `과학`
  - `경제`
  - `고사성어`
  - `지리`

### 역할
- 필수 단어 추천 후보군 제한
- 랜덤 단어 후보군 제한
- connector word 후보군 제한

### 특별 규칙
- `고사성어`는 필수 단어 4개 중 **1개 반드시 포함**
- 따라서 고사성어 비활성 상태에서는 저장 불가 또는 경고 처리 필요

---

## 1.6 허용 등급
- 항목명: `허용 등급`
- 입력 방식: 멀티 선택 가능한 dropdown select box
- 선택값:
  - `1`
  - `2`
  - `3`
  - `4`

### 역할
- 사전에서 선택 가능한 단어의 등급 필터
- 필수 단어 / 랜덤 단어 / connector word 후보군 제한 기준

### 예시
- 초반 스테이지: `1, 2`
- 중반 스테이지: `2, 3`
- 후반 스테이지: `3, 4`

### 비고
초기 버전에서는 하나의 공통 등급 필터만 둔다.
향후 필요 시 아래처럼 세분화 가능하다.

- 필수 단어 허용 등급
- 랜덤 단어 허용 등급
- connector word 허용 등급

---

## 1.7 버튼 영역

### 추천하기
- 역할:
  - 현재 입력 조건을 기준으로 필수 단어 4개를 추천
  - 추천 결과는 가운데 패널에 표시

### 다시 추천
- 역할:
  - 현재 입력 조건은 유지한 채 추천 단어만 다시 생성
  - seed를 유지할지 변경할지는 향후 정책 결정 가능

### stage.xml 저장
- 역할:
  - 현재 확정된 단어 조합과 설정을 기반으로 `stage.xml` 생성

### 저장 전 검증
저장 전 다음 조건을 만족해야 한다.

- 필수 단어 수가 정확히 4개인가
- 고사성어 1개가 포함되어 있는가
- 4개 단어가 2개 그룹으로 구성 가능한가
- 각 그룹 내부에 실제 교차 가능한 글자가 있는가
- 허용 등급을 위반하지 않는가

---

# 2. 가운데 패널: 필수 단어 추천 결과

## 2.1 목적
이 패널은 추천된 필수 단어 4개를 **2개 그룹**으로 보여준다.

각 그룹은 다음 조건을 만족해야 한다.

- 단어 2개로 구성
- 두 단어가 실제 크로스워드에서 교차 가능
- 단순 문자 포함이 아니라 실제 교차 가능한 동일 글자 존재

---

## 2.2 그룹 표시 구조
- `그룹 A`
- `그룹 B`

각 그룹은 카드 형태로 표시한다.

### 그룹 상단 표시 정보
- 그룹 이름 (`A`, `B`)
- 추천 교차 글자
- 필요 시 연결 안정성 표시
  - 높음
  - 보통
  - 낮음

---

## 2.3 단어 카드 항목
각 단어 카드에는 다음 정보가 표시된다.

### 표시 항목
- 단어
- 카테고리
- 등급
- 뜻풀이

### 예시
- `타산지석`
- `고사성어`
- `등급 3`
- `다른 사람의 허물을 자신의 거울로 삼음.`

---

## 2.4 단어 카드 동작
각 단어 카드에는 다음 조작 기능이 필요하다.

### 고정
- 역할:
  - 현재 단어를 유지 대상으로 지정
  - 이후 다시 추천 시 이 단어는 유지하고 나머지만 교체

### 교체
- 역할:
  - 전체를 다시 추천하지 않고 현재 단어만 새로운 후보로 바꿈

---

## 2.5 필수 단어 추천 규칙
추천 엔진은 다음 조건을 만족하는 4개 단어를 제안해야 한다.

### 공통 규칙
- 총 4개 단어 추천
- 2개씩 2그룹 구성
- 최소 1개는 `고사성어`
- 허용 카테고리 안에서만 선택
- 허용 등급 안에서만 선택

### 그룹 규칙
- 그룹 A: 2개 단어
- 그룹 B: 2개 단어
- 각 그룹 내부는 실제 교차 가능해야 함

### 중요
다음은 **교차 가능**으로 인정하지 않는다.

- 단어 내부에 같은 글자가 여러 번 있다는 이유만으로 그룹 인정
- 눈에 보이는 문자가 비슷하다는 이유만으로 그룹 인정
- 실제 격자 교차 불가능한 조합

즉 추천 엔진은 단순 문자열 포함이 아니라 **실제 교차 가능성 검증**을 수행해야 한다.

---

# 3. 오른쪽 패널: 자동 검증 및 XML 요약

## 3.1 자동 검증 영역
추천된 조합이 스테이지 조건에 맞는지 보여주는 영역이다.

### 표시 항목 예시
- 필수 단어 수: `4 / 4`
- 고사성어 포함 여부: `예 / 아니오`
- 허용 등급 충족 여부
- 목표 플레이 단어 수: `9`
- 자동 범위: `8 ~ 10`
- 그룹 연결성: `양호 / 보통 / 낮음`
- 예상 connector word 수: `3 ~ 5`

---

## 3.2 메모 / 경고 영역
자동 검증 결과 외에 디자이너가 주의해야 할 내용을 보여준다.

### 예시
- connector word가 많이 필요할 수 있음
- 긴 단어 비중이 높아 생성 실패 가능성이 있음
- 허용 등급에 비해 고사성어 난이도가 높음
- 특정 그룹이 약하게 연결되어 있음

---

## 3.3 stage.xml 미리보기 영역
현재 설정값을 바탕으로 생성될 `stage.xml`의 요약 내용을 표시한다.

### 목적
- 저장 전에 구조를 빠르게 검토
- stage 파일에 어떤 정보가 들어가는지 확인

### 표시 예시
- stage id / level
- board rows / columns
- playWords target / range
- grades
- requiredGroups
- randomWordPolicy
- connectorWordPolicy
- seedPolicy

---

# 4. 내부 추천 규칙 요약

## 4.1 필수 단어 수
- 항상 `4개`

## 4.2 필수 단어 비율
- 플레이 단어 기준 약 `40%`

## 4.3 고사성어 규칙
- 필수 단어 4개 중 `1개 반드시 포함`

## 4.4 랜덤 단어 규칙
- seed 기반 랜덤 단어 `1개`
- 이 단어는 stage 시작 시 최종 퍼즐 생성에 사용

## 4.5 connector word 규칙
- 남은 단어는 connector word로 채움
- 목표 수는 플레이 단어 수와 랜덤 단어 수를 고려해 자동 계산

예:
- 플레이 단어 목표값: `9`
- 필수 단어: `4`
- 랜덤 단어: `1`
- connector word 예상: `4`

실제 범위는 `3 ~ 5` 정도 허용 가능

---

# 5. 입력값과 출력값 정리

## 입력값
- 스테이지 번호
- seed (`0 ~ 100`)
- 행 수
- 열 수
- 플레이 단어 목표값
- 사용 카테고리 목록
- 허용 등급 목록

## 출력값
- 필수 단어 4개
- 그룹 A / 그룹 B 구성
- 자동 검증 결과
- `stage.xml` 미리보기
- 최종 `stage.xml` 파일

---

# 6. 향후 확장 가능 항목
초기 버전에는 넣지 않지만, 추후 확장 가능한 항목은 다음과 같다.

- 단어별 개별 고정/교체 강화
- connector word 미리보기
- 레이아웃 생성 가능성 점수
- 실제 미니 보드 미리보기
- 랜덤 단어 후보 미리보기
- 그룹 간 bridge connector 후보 표시
- 필수 단어/connector word 등급 정책 분리

---

# 7. 결론
이 레벨디자인 프로그램은 다음에 집중한다.

- 스테이지를 빠르게 설계할 수 있어야 함
- 필수 단어 4개를 자동 추천할 수 있어야 함
- 추천 결과가 실제 교차 가능한지 검증해야 함
- 최종적으로 `stage.xml`을 안정적으로 생성해야 함

즉, 이 프로그램은 퍼즐 보드 에디터가 아니라 **필수 단어 추천 및 stage.xml 생성 도구**로 정의한다.
