# 레벨디자인 프로그램 화면 항목 명세서

## 문서 목적
이 문서는 한국어 십자말 풀이의 `stage.xml` 파일을 생성하기 위한 레벨디자인 프로그램의 화면 구성과 입력/출력 항목을 정의한다.

이 프로그램의 목적은 퍼즐 배치를 직접 설계하는 것이 아니라,

- 스테이지 조건 입력
- 필수 단어 4개 추천
- 추천 결과 검토
- `stage.xml` 생성

을 빠르고 안정적으로 수행하도록 돕는 것이다.

---

## 프로그램 역할 요약
1. `사전.v0001.csv`를 읽는다.
2. 디자이너가 입력한 조건에 맞는 필수 단어 4개를 추천한다.
3. 추천 단어가 실제로 2개 그룹으로 교차 가능한지 검증한다.
4. 검토가 완료되면 `stage.xml`을 생성한다.
5. `stage.xml`에는 필수단어와 생성 조건만 저장하고, 연결단어 / leaf단어 수는 미리 저장하지 않는다.

---

## 전체 화면 구조
- 왼쪽 패널: 스테이지 설정
- 가운데 패널: 필수 단어 추천 결과
- 오른쪽 패널: 자동 검증 및 `stage.xml` 요약

---

# 1. 왼쪽 패널: 스테이지 설정

## 1.1 스테이지 번호
- 항목명: `시작 스테이지 번호`
- 입력 방식: 숫자 입력
- 역할:
  - `stage.xml`의 `level` / `id` 생성 기준
  - 단일 저장 파일명 생성 기준
  - ZIP 일괄 생성 시작 번호 기준

### 제약
- 정수만 허용
- 최소값: `1`

---

## 1.2 Seed
- 항목명: `Seed`
- 입력 방식: 숫자 입력
- 역할:
  - 랜덤 단어 선택
  - 퍼즐 생성 시도 순서
  - 같은 stage에서 재현 가능한 결과 보장

### 제약
- 허용 범위: `0 ~ 100`

---

## 1.3 보드 크기
- 항목명: `보드 크기`
- 입력 방식: 숫자 입력 2개 (`행`, `열`)
- 역할:
  - 스테이지 퍼즐 생성에 사용할 보드 크기 지정

---

## 1.4 플레이 단어 수
- 항목명: `플레이 단어 수`
- 입력 방식: 숫자 입력 1개
- 역할:
  - 실제 스테이지에서 사용되는 전체 단어 수의 목표값

### 자동 규칙
입력값은 목표값으로만 사용한다.

예:
- 입력값 `9`
- 내부 적용 범위 `8 ~ 10`

즉 시스템은 자동으로 다음을 계산한다.
- `min = target - 1`
- `target = 입력값`
- `max = target + 1`

---

## 1.5 사용 카테고리
- 항목명: `사용 카테고리`
- 입력 방식: 다중 선택
- 카테고리 목록:
  - `일상`
  - `과학`
  - `경제`
  - `지리`
  - `고사성어`

### 역할
- 필수 단어 후보군 제한
- 랜덤 단어 후보군 제한
- 추가 단어 후보군 제한

### 특별 규칙
- `고사성어`는 필수 단어 4개 중 1개 반드시 포함되므로 항상 포함되어야 한다.

---

## 1.6 허용 등급
- 항목명: `허용 등급`
- 입력 방식: 멀티 선택
- 선택값: `1`, `2`, `3`, `4`

### 역할
- 필수 단어 / 랜덤 단어 / 추가 단어 후보군 제한 기준

---

## 1.7 버튼 영역

### 추천하기
- 현재 입력 조건을 기준으로 필수 단어 4개를 추천한다.

### 다시 추천
- 현재 입력 조건은 유지한 채 추천 단어만 다시 생성한다.

### 현재 stage.xml 저장
- 현재 확정된 단어 조합과 설정을 기반으로 단일 `stage.xml`을 저장한다.

### 10개 ZIP 다운로드
- 시작 스테이지 번호부터 `+1`씩 증가하는 10개 `stage.xml`을 생성해 ZIP으로 저장한다.

### 저장 전 검증
저장 전 다음 조건을 만족해야 한다.
- 필수 단어 수가 정확히 4개인가
- 고사성어 1개가 포함되어 있는가
- 4개 단어가 2개 그룹으로 구성 가능한가
- 각 그룹 내부에 실제 교차 가능한 글자가 있는가
- 허용 등급을 위반하지 않는가

---

# 2. 가운데 패널: 필수 단어 추천 결과

## 목적
추천된 필수 단어 4개를 2개 그룹으로 보여준다.

각 그룹은 다음 조건을 만족해야 한다.
- 단어 2개로 구성
- 두 단어가 실제 크로스워드에서 교차 가능

## 단어 카드 표시 항목
- 단어
- 카테고리
- 등급
- 뜻풀이

## 단어 카드 동작
### 고정
- 이후 다시 추천 시 현재 단어를 유지

### 교체
- 전체를 다시 추천하지 않고 현재 단어만 교체

---

# 3. 오른쪽 패널: 자동 검증 및 XML 요약

## 3.1 자동 검증 영역
표시 항목 예시
- 필수 단어 수: `4 / 4`
- 고사성어 포함 여부: `예 / 아니오`
- 허용 등급 충족 여부
- 목표 플레이 단어 수: `9`
- 자동 범위: `8 ~ 10`
- 그룹 연결성: `양호 / 재검토 필요`

## 3.2 메모 / 경고 영역
표시 예시
- 현재 결과는 Seed + Stage 기준 추천 조합
- 연결단어와 leaf단어 수는 생성 시 실제 배치 결과에 따라 결정됨
- 특정 그룹이 약하게 연결되어 있으면 교체 권장

## 3.3 stage.xml 미리보기 영역
표시 항목 예시
- stage id / level
- board rows / columns
- playWords target / range
- grades
- requiredGroups
- randomWordPolicy
- seedPolicy

---

# 4. 내부 추천 규칙 요약

## 4.1 필수 단어 수
- 항상 `4개`

## 4.2 고사성어 규칙
- 필수 단어 4개 중 `1개 반드시 포함`

## 4.3 랜덤 단어 규칙
- seed 기반 랜덤 단어 `1개`
- 이 단어는 stage 시작 시 최종 퍼즐 생성에 사용

## 4.4 연결단어 / leaf단어 규칙
- 이 둘은 `stage.xml`에 미리 저장하지 않는다.
- 최종 생성된 퍼즐에서 각 단어의 교차수에 따라 판정한다.
  - 교차수 `2 이상`: 연결단어
  - 교차수 `1`: leaf단어

---

# 5. 입력값과 출력값 정리

## 입력값
- 시작 스테이지 번호
- seed (`0 ~ 100`)
- 행 수
- 열 수
- 플레이 단어 목표값
- 사용 카테고리 목록
- 허용 등급 목록

## 출력값
- 필수 단어 추천 결과
- 자동 검증 결과
- `stage.xml` 미리보기
- 단일 `stage.xml` 파일
- 10개 `stage.xml` ZIP 파일
